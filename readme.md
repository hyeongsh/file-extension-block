# 파일 확장자 차단 과제 요구조건

## 1. 과제 목표
1-1. 고정 확장자는 차단을 자주하는 확장자 리스트이며, default는 unCheck되어져 있도록 한다. <br>
1-2. 고정 확장자를 check or uncheck 할 경우 db에 저장되도록 한다. -> 새로고침 시 유지될 db에 저장한다. <br>

2-1. 확장자 최대 입력 길이는 20자리이다. <br>
2-2. 추가 버튼 클릭 시 db에 저장되며, 아래쪽 영역에 표현된다. <br>

3-1. 커스텀 확장자는 최대 200개까지 추가가 가능하다. <br>
3-2. 확장자 옆 X를 클릭할 경우 db에서 삭제된다. <br>

* 추가
- Enter 키 입력으로도 커스텀 확장자 추가가 가능하도록 하였습니다.
- 영문/숫자 외에 다른 문자가 포함된 확장자의 경우 잘못된 확장자로 판단하고 예외 처리하였습니다.
- 대문자 입력의 경우 윈도우 기준 확장자의 대소문자를 구분하지 않으므로, OS 호환을 위해 소문자 처리하여 통일하였습니다.
- 고정 확장자를 커스텀 확장자에 입력하여 차단하려는 경우, 책임 범위 상 잘못된 동작이라 판단하고 예외 처리하였습니다.

## 2. 구현 범위
- 사용자 인증/권한 구분은 하지 않을 것.
- 관리자 대상으로 확장자 차단을 관리하는 화면과 API를 구현한다.

## 3. 화면상의 요구조건 (Frontend)
### 3.1 기본 화면
- 제목: `파일 확장자 차단`
- 설명 문구: 확장자별 첨부/전송 제한 목적 안내
- 파일은 `static` 경로에서 정적 리소스로 제공한다.

### 3.2 고정 확장자 영역
- 체크박스 기반으로 `bat`, `cmd`, `com`, `cpl`, `exe`, `scr`, `js`를 나열한다.
- 페이지 진입 시 서버에서 현재 차단 상태를 조회해 체크 상태를 동기화한다.
- 체크 변경 시 서버에 즉시 반영 요청을 보낸다.

### 3.3 커스텀 확장자 영역
- 텍스트 입력 + `+추가` 버튼으로 확장자를 추가한다.
- Enter 키 입력으로도 추가가 가능하게 만든다.
- 목록은 각 항목에 `X` 삭제 버튼을 제공한다.
- 페이지 진입/추가 후 목록을 서버 기준 데이터로 다시 로드한다.
- 화면에 `현재개수 / 200` 카운터를 표시한다.

## 4) 서버 요구조건 (Backend)
### 4.1 고정 확장자 조회
- `GET /fixed`
- 고정 확장자 목록과 `blocked` 상태를 반환한다.

### 4.2 고정 확장자 차단 상태 변경
- `PATCH /fixed`
- 요청 바디: `extension`, `block`
- 대상 확장자가 존재하고 `FIXED` 타입일 때만 상태를 변경한다.

### 4.3 커스텀 확장자 조회
- `GET /custom`
- 커스텀 확장자 목록을 반환한다.

### 4.4 커스텀 확장자 추가
- `POST /custom`
- 요청 바디: `extension`
- 정상 추가 시 `201 Created`를 반환한다.

### 4.5 커스텀 확장자 삭제
- `DELETE /custom?extension={name}`
- 정상 삭제 시 `204 No Content`를 반환한다.

## 5) 데이터 및 DB 요구조건
- DB는 SQLite를 사용한다. (`jdbc:sqlite:mydb.sqlite`)
- 엔티티 `extensions` 테이블 구조:
  - `name` (PK) -> 문자열로 구성
  - `extension_type` (`FIXED` / `CUSTOM`)
  - `blocked` (boolean)
- JPA `ddl-auto: update` 설정으로 스키마를 유지하며, 추후 필요하다면 `validate`로 변경한다.

## 6) 입력 검증
- `extension`이 `null`이면 오류
- 공백 문자열이면 오류
- 길이 20자 초과면 오류
- 영문/숫자 외 문자가 포함되면 오류
- 저장/조회 전에 소문자로 변환한다.

## 7) 중복/충돌 처리 요구조건
- 이미 `FIXED`에 존재하는 확장자를 커스텀으로 추가하면 충돌 오류
- 이미 `CUSTOM`에 존재하는 확장자를 다시 추가하면 충돌 오류
- 고정 확장자 API에서 `FIXED`가 아닌 대상을 수정하려고 하면 오류
- 존재하지 않는 확장자를 삭제하려 하면 오류
- 200개가 넘는 커스텀 확장자를 넣으려고 하면 오류

## 8) 오류 응답 요구조건
- 오류 응답 형식: `{ "message": "..." }`
- 주요 상태 코드:
  - `400 Bad Request`: 잘못된 확장자, 고정 확장자 아님
  - `409 Conflict`: 이미 차단됨, 고정 목록에 이미 존재함

## 9) 테스트 요구조건
- 서비스 레이어 테스트에서 아래 시나리오를 검증한다.
  - 고정 확장자 상태 변경 성공/실패
  - 커스텀 확장자 추가 실패 케이스(빈 값, 20자 초과, 중복, 고정 확장자 충돌)
  - 커스텀 확장자 추가 초과 케이스
  - 커스텀 확장자 삭제 성공/실패
